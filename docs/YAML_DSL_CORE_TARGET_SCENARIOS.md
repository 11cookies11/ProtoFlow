# YAML-DSL 核心功能目标工况（已确认）

日期：2026-03-06  
状态：已确认，作为核心功能范围基线

## 1. 调试交互类
- CLI/AT 命令调试：模块（Wi-Fi/4G/BT/GNSS）AT 指令，发命令 -> 看回显 -> 改参数。
- 功能开关/模式切换：进入工厂模式、打开日志、切换调试级别、开启某功能。
- 现场问题复现脚本：一键执行“复现路径”，把关键回显打包给他人。

## 2. 升级与引导类
- 进入 Bootloader / 选择启动模式：命令触发、握手口令、超时窗口。
- 固件升级/补丁下发：X/YMODEM、自定义分包、擦除/写入/校验。
- 版本一致性校验：升级后自动读取版本/CRC/分区信息并确认成功。

## 3. 参数配置与产线写入
- SN/设备ID 写入（并读回校验）。
- 校准数据写入：ADC 偏移、传感器标定、温漂补偿。
- 密钥/证书注入：设备认证用途（要求权限控制与审计）。
- 区域/法规配置：例如射频 Region、功率限制。

## 4. 产测/老化/自检（ATE）
- 一键自检流程：执行指令序列并采集传感器/电机/继电器状态。
- 阈值断言：温度/电压/电流/转速等范围判定 PASS/FAIL。
- 老化测试：循环执行、定时采样，异常即停机并打点日志。
- 测试报告导出：CSV/JSON/HTML。

## 5. 日志与故障诊断
- 抓取运行日志/崩溃信息：保存到文件并自动标记关键行。
- 错误码解释/聚合：`err=xxx` 映射原因（可维护映射表）。
- 现场采集包：版本 + 配置 + 最近日志 + 设备信息，一键导出。

## 6. 自动化桥接场景（串口只是入口）
- 串口 + 外部命令：刷机后自动运行 PC 端脚本并生成产物。
- 串口 + 网络接口：串口完成网络参数配置后，切换 TCP/HTTP 继续测试。
- 多设备/多串口：单工装覆盖多台设备（可先队列串行，后续并行）。

## 说明
- 本文档用于固定 YAML-DSL 核心功能目标工况，不包含实现细节与优先级拆分。
- 具体语法、动作、验收矩阵在后续设计文档中落地。
