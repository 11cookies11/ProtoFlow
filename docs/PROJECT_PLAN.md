# ProtoFlow 项目计划书（执行版）

> 项目属性：个人项目（由你主导实施，Codex 辅助实现与落盘）。

## 1. 项目目标

- 将 ProtoFlow 从“可演示”推进到“可稳定交付”版本。
- 打通核心链路：连接设备 -> 收发数据 -> 协议解析 -> 脚本执行 -> 可视化监控。
- 发布一个可安装、可验证、可扩展的 `v1.0`。

## 2. 当前现状（简述）

- 已有：Web UI、Qt WebChannel 桥接、串口/TCP 通信、DSL 执行框架、插件加载框架。
- 待完善：部分功能仍演示化（如部分监控/图表 mock）、模块化能力与测试覆盖不足、端到端稳定性待提升。
- 当前阶段策略：`COM(Serial) 优先`，TCP 相关开发暂缓（仅保留已有能力，不做新功能扩展）。

## 3. 范围定义

### In Scope

1. 通信核心稳定化（COM/Serial、异常恢复、日志）
2. DSL 脚本执行闭环（运行、停止、状态反馈）
3. 协议解析能力增强（最少覆盖 Modbus RTU + 可扩展协议）
4. UI 与后端能力对齐（去除假状态、补齐真实数据）
5. 打包发布与文档完善

### Out of Scope（本期不做）

1. 全量 Web 可视化流程编辑器
2. 大规模云端协同功能
3. TCP 新能力开发与专项优化（延后到下一阶段）

## 6. 验收标准（量化）

1. COM/Serial 连接成功率 > 99%（标准测试环境）。
2. 核心场景连续运行 8 小时无崩溃。
3. DSL 脚本关键流程通过率 100%。
4. UI 核心页面无占位交互、无阻塞级缺陷。
5. 发布包可一键安装并完成基础验证。

## 7. 风险与应对

1. 协议差异导致解析不稳定  
   处理：建立协议样本库 + 回归用例。
2. UI 与后端接口频繁变动  
   处理：冻结桥接接口版本，变更走评审。
3. 设备环境不可控  
   处理：增加模拟器与录包回放测试。
